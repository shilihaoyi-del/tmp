# 基于SC171V2的远程体感机械臂控制与可视化展示系统项目计划书

## 一、项目名称

基于SC171V2与人体动作识别的远程机械臂控制及可视化展示系统

 具体目标

1. 在电脑端完成摄像头图像采集和人体姿态识别。
2. 将人体肩部、肘部、手腕和手指动作映射为机械臂关节控制数据。
3. 通过Mosquitto MQTT Broker实现电脑端与SC171V2之间的远程通信。
4. 由SC171V2接收控制指令，完成数据校验、运行状态管理。
5. 由STM32完成幻尔总线舵机控制。
6. 实现机械臂对人体动作的基本同步跟随。
7. 开发网页实时展示机械臂六关节角度、在线状态和三维运动情况。
8. 实现断线保持、超时停止、关节限位和急停等安全功能。
9. 完成适用于比赛现场的系统部署和演示流程。


 系统组成与设备分工

### 5.1 电脑端

电脑端主要承担人体动作识别、关节角度计算和远程控制指令发送。

主要功能包括：

1. 读取USB摄像头视频。
2. 识别手部关键点。
3. 计算人体关节角度。
4. 对识别结果进行滤波和异常值处理。
5. 将人体动作映射为机械臂目标角度。
6. 通过MQTT发送控制数据。
7. 运行本地网页，展示机械臂状态和运动情况。
8. 提供启动、暂停和急停等操作按钮。


服务器主要功能包括：

1. 接收电脑端发送的机械臂控制指令。
2. 将控制指令转发给SC171V2。
3. 接收SC171V2上传的设备状态。
4. 将设备状态转发给网页。
5. 管理MQTT设备连接和主题权限。
6. 提供MQTT普通连接和MQTT WebSocket连接。
7FastAPI
8服务器上部署网页

SC171V2主要功能包括：

1. 通过5G或Wi-Fi连接公网服务器。
2. 订阅机械臂控制主题。
3. 解析电脑端发送的六关节目标数据。
4. 检查数据序号、时间戳和有效期。
5. 对机械臂关节角度进行安全限位。
6. 检查电脑端和服务器的心跳状态。
7. 将合法控制指令发送给STM32。
8. 接收STM32返回的状态信息。
9. 向服务器上传机械臂运行状态。
10. 在网络异常时进入保持或停止状态。


STM32主要负责机械臂实时执行和舵机通信。

1. 接收SC171V2发送的六关节目标值。
2. 校验串口数据帧和CRC。
3. 对目标角度进行限幅。
4. 对舵机运动进行插值和平滑处理。
5. 限制关节最大速度和加速度。
6. 生成幻尔总线舵机控制指令。
7. 同步控制六个机械臂舵机。
8. 检测SC171V2通信超时。
9. 接收实体急停输入。
10. 在条件允许时读取舵机位置。

 
人体动作识别方案
7.1 识别内容

电脑端通过摄像头识别以下人体关键点：

手腕

根据关键点计算：

1. 肩部抬起角度。
2. 肘部弯曲角度。
3. 手腕上下运动趋势。
4. 手掌方向。
5. 拇指与食指之间距离。

### 7.2 动作映射

建议第一阶段采用关节直接映射方式。

| 人体动作    | 机械臂动作 |
| ------- | ----- |
| Swipe Right Swipe Left  | 底座旋转  |
| Swipe Down Swipe Up(右手)  | 肩关节运动 |
| Swipe Down Swipe Up(左手)| 肘关节运动 |
| Swipe Down Swipe Up(左手右手一起) | 腕部俯仰  |
| Swipe V (V形滑动)(左手) | 腕部旋转  |
| Pinch  Grab| 夹爪闭合  |
| Expand | 夹爪张开  |

注意事项
总共14个分类器,上面提到的处理,不提到的不处理,夹爪闭合的pinch和Grab识别到一种就行

建议人体动作识别数据发送频率为20至30 Hz。


网页主要作为比赛现场的实时展示界面，不承担核心运动控制。

比赛电脑本地运行Vue网页，通过Mosquitto WebSocket端口获取机械臂实时状态。


UI主页面展示：

1. 系统名称和项目标志。
2. SC171V2在线状态。
3. STM32在线状态。
4. 机械臂当前运行模式。
5. 六个关节目标角度。
6. 六个关节实际角度或预测角度。
7. 网络通信延迟。
8. 控制数据发送频率。
9. 机械臂三维模型。(有成品建模)
10. 实时关节变化曲线。
11. 故障和急停状态。
12. 启动、暂停和急停按钮。
