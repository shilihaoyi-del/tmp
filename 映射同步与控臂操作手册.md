# JetArm 机械臂 ↔ 网页映射同步与控臂操作手册

本文用于：**先确认机械臂与网页 3D 是否同步**，再进行日常控臂。请按顺序做，不要跳步。

---

## 0. 设备与角色（先搞清谁干什么）

| 角色 | 地址 / 位置 | 作用 |
|------|-------------|------|
| 观摩网页 | `http://121.41.67.80:8000/` | 显示 3D 模型与关节角 |
| 云端 | `121.41.67.80` | MQTT / API 中转，**START 后才转发运动** |
| AidLux（广和通 SC171） | `192.168.42.4` 用户 `aidlux` / 密码 `aidlux` | USB 连机械臂，跑舵机桥接 |
| 电脑视觉 | 本机摄像头 + `hand_recognition_ddnet.py` | 识别手势；默认**不驱动**，按 `s` 才控臂 |

**重要：**

- 只开网页、不开 AidLux 桥接 → 网页画的是 **home 示意（Z 轴直线）**，不是你手摆的实时姿态。
- 要验证「实物 ↔ 网页」同步，**必须开桥接**，让网页收到 `actual`（实测角度）。

---

## 1. 安全约定（每次测试前）

1. 确认机械臂周围无障碍、下方无人手。
2. 需要徒手摆姿态时：**先卸力（取消使能）**，再动手。
3. 不要同时开：`~/Desktop/bridge` 与 `free_move_*`（会抢 CH340）。
4. 视觉开着但未按 `s` / 网页未点 START → 臂不应被手势驱动。
5. 异常立刻：网页点急停，或视觉窗口按 `e`，或停掉桥接并卸力。

## 2. 当前 Home 与网页映射（已标定）

**Home（激活 / 网页 Z 线参考）关节角（度）：**

```text
[base, shoulder, elbow, wrist_p, wrist_r, gripper]
[-0.72, -87.6, -36.72, -183.0, -4.8, 45.0]
```

文件：`sc171v2_agent/home_pose.json`（AidLux 上同样有一份）。

**网页映射规则：**

- 当关节角 = 上述 home 时，3D 模型 j1～j5 显示为约 **0°**，整体呈 **Z 轴一直线**。
- 有设备上报时，网页优先显示 **actual（实测）**；无设备时显示 home 示意。

## 3. 映射同步验证（请仔细按步做）

目标：确认「你看到的实物姿态」和「网页 3D」一致。

### 步骤 A — 卸力（取消使能）

在 **AidLux 终端**执行（或让助手帮你卸力）：

```bash
# 停桥接（若在跑）
echo aidlux | sudo -S -p '' pkill -f sc171v2_servo_bridge.py || true

# 若桌面有 bridge 脚本，也先不要双击启动
```

卸力后舵机应可徒手扳动。若不确定是否已卸力，可让助手执行「取消使能」。

### 步骤 B — 徒手摆到 Home（或你要对照的姿态）

对照上一节 Home 角度，尽量摆接近：

| 关节 | Home 约值 | 徒手感觉（参考） |
|------|-----------|------------------|
| base | ≈ 0° | 底座接近正中 |
| shoulder | ≈ -88° | 接近肩部中位 |
| elbow | ≈ -37° | 不要伸直成 0° |
| wrist_p | ≈ -183° | 腕俯仰较靠限位一侧 |
| wrist_r | ≈ -5° | 接近中位 |
| gripper | （常读不到） | 可先不管 |

摆好后手离开臂，准备开桥接。

### 步骤 C — 启动舵机桥接（开始上报 actual）

在 **AidLux** 上：

```bash
~/Desktop/bridge
```

或：

```bash
bash ~/Desktop/start_gesture_bridge.sh
```

等待日志出现类似：

- `[OK] MQTT ...`
- `H0-HOME`（会尝试回到 home；若已在 home 附近，变化应很小）
- 之后持续有位置读取 / 状态上报

**说明：** 桥接启动会 **LOAD（使能）** 并执行回 home。若你只想读位置、不要自动回 home，先告诉助手改流程；默认手册按「开桥接 = 可同步网页」处理。

### 步骤 D — 打开 / 刷新观摩页

1. 浏览器打开：`http://121.41.67.80:8000/`
2. 强制刷新：`Ctrl + F5`（避免旧 JS 缓存）
3. 看页面状态：
   - 设备 / 桥接相关指示应为在线（BRIDGE UP / device 有数据）
   - 关节数字应为 **实测**，不是全 0

### 步骤 E — 肉眼对照（核心）

站在机械臂侧面，同时看网页 3D：

1. **静止对照**  
   - 底座朝向是否一致  
   - 大臂 / 小臂弯曲是否一致  
   - 腕俯仰、腕滚转是否一致  

2. **微动对照（推荐）**  
   - 若当前已使能：用网页 **START** 后小范围控一下（或暂时卸力徒手微扳再重新开桥接读数）  
   - 更稳妥：保持桥接运行，用视觉按 `s` 后做一个小手势，看网页是否同向、同轴变化  

3. **判定**

| 现象 | 结论 |
|------|------|
| 静止姿态一致，微动同向 | **已同步**，可进入日常控臂 |
| 静止差很多，但桥接在线 | 实物不在 home，或映射轴号/正负有问题 → 记录哪一轴反了 |
| 网页一直是 Z 直线、数字不变 | 桥接没连上云端 → 查 AidLux 网络 / MQTT / 日志 |
| 某一轴上下反了 | 只需改网页 `VIEW_JOINT_SIGN`，一般**不必重标 home** |

把「哪一轴不对、是反了还是差多少」记下来，便于改映射。

### 步骤 F — 验证完后卸力（建议）

对照结束后若要徒手再摆或离开：

- 停桥接并卸力（可再说「取消使能」）
- 或网页急停 / 视觉 `e` 后停桥接

## 4. 日常控臂流程（同步确认之后）

### 4.1 AidLux：开桥接

```bash
~/Desktop/bridge
```

等 MQTT 就绪、回 home 完成。

### 4.2 电脑：开视觉（默认不动臂）

```bat
一键启动手势控臂.bat
```

或在项目目录：

```bat
set ARM_API_BASE=http://121.41.67.80:8000
python hand_recognition_ddnet.py
```

窗口应显示类似：`ARM:OFF(按s启动)`。

此时可以挥手识别，**手臂不应跟着动**。

### 4.3 开始驱动（二选一）

- 视觉窗口按键盘 **`s`**  
- 或观摩页点 **START**

显示变为 `ARM:ON` 后，手势才会驱动机械臂。

### 4.4 常用键

| 键 | 作用 |
|----|------|
| `s` | START，允许驱动 |
| `p` | PAUSE，暂停驱动 |
| `e` | 急停 |
| `q` | 退出视觉 |

### 4.5 结束

1. 视觉 `e` 或网页急停  
2. 退出视觉 `q`  
3. AidLux 停桥接并卸力（需要徒手或离开时）

---

## 5. 速度与消抖（已配置，一般不用改）

- 舵机 `move_time_ms = 1000`（约为原 500ms 的一半速度）
- 正常运动：雅可比平滑消抖（`H1-JAC`）
- **回 home：关闭雅可比**，保证能到位（`H0-HOME` / `home_exact`）

---

## 6. 故障速查

| 现象 | 处理 |
|------|------|
| 网页不跟着动 | AidLux 是否运行 `~/Desktop/bridge`；本机能否访问 `121.41.67.80`；`Ctrl+F5` |
| 一开视觉臂就动 | 应已修复；确认用的是最新脚本，且未误按 `s`；云端需已部署「不自动 START」 |
| 开桥接臂猛动 | 正常会回 home；先把臂大致摆到 home 再开桥接更稳 |
| 某一轴网页反了 | 记录轴名，改前端 `VIEW_JOINT_SIGN`（不必重标整个 home） |
| CH340 / 打不开串口 | 确认没有第二个程序占用；停掉 `free_move_*` |
| 夹爪无读数 | 常见，可先忽略；home 夹爪用 45° |

## 7. 本次「映射同步」推荐清单（打印用）

- [ ] 卸力，周围清空

- [ ] AidLux：`~/Desktop/bridge`

## 8. 和助手配合的常用口令

- 「取消使能」→ 停桥接 + UNLOAD  
- 「回 home」→ 使能并走到 home  
- 「记录」→ 读取当前姿态写入 home（会改映射零点时需同步说清）  
- 「某轴反了」→ 只调网页符号，不重摆整臂
