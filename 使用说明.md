# 使用说明 — Fibocom SC171V2 远程体感机械臂

本文说明如何在本地电脑跑通：**观摩网页 + 云桥后端 +（可选）摄像头手势控臂**。

主角是 **广和通 SC171V2**；手势识别与网页是场景包装。硬件未接时可用模拟器演示。

---

## 一、目录结构（你需要关心的）

```
hand-recognition/
├── backend/                 # FastAPI + MQTT 桥 + 模拟器
├── frontend/                # Vue 观摩台
├── hand_recognition_ddnet.py   # 摄像头识别
├── pc_arm_bridge.py            # 手势→关节→HTTP/MQTT
├── requirements-vision.txt
└── 使用说明.md              # 本文件
```

---

## 二、两种运行模式

| 模式 | 适用 | 后端模拟器 | 摄像头脚本 |
|------|------|------------|------------|
| **A. 纯模拟演示** | 先看界面、不接摄像头 | `ENABLE_SIMULATOR=true` | 不跑 |
| **B. 真摄像头联调** | 手势驱动关节 | `ENABLE_SIMULATOR=false` | 跑 `hand_recognition_ddnet.py` |

两种模式都要先开后端；前端可用 Vite，也可直接打开后端托管的页面。

### 4）操作说明（识别窗口）

| 按键 | 作用 |
|------|------|
| 做手势（凑满 32 帧） | 映射关节并 `POST /api/cmd` |
| `q` | 退出 |
| `s` | 重新 start |
| `p` | pause |
| `e` | 急停 |

终端出现 `[publish #N] ...` 即表示已发给后端；网页关节应跟随变化。

### 5）手势映射（计划书）

| 手势 | 机械臂 |
|------|--------|
| Swipe Right / Left | 底座 |
| Swipe Up / Down（右手） | 肩 |
| Swipe Up / Down（左手） | 肘 |
| 双手同时上下滑 | 腕俯仰 |
| Swipe V（左手） | 腕旋转 |
| Pinch / Grab | 夹爪闭合 |
| Expand | 夹爪张开 |

其余分类器忽略。

### 6）环境变量（可选）

| 变量 | 默认 | 含义 |
|------|------|------|
| `ARM_API_BASE` | `http://127.0.0.1:8000` | 后端地址 |
| `ARM_USE_HTTP` | `1` | 用 HTTP 发指令 |
| `ARM_USE_MQTT` | `0` | 设为 `1` 同时发 MQTT（需 Mosquitto） |
| `ARM_MQTT_HOST` | `127.0.0.1` | MQTT 主机 |
| `ARM_MQTT_PORT` | `1883` | MQTT 端口 |

示例（同时开 MQTT）：

```powershell
$env:ARM_USE_MQTT="1"
python hand_recognition_ddnet.py
```

## 六、常用 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/health` | 健康检查 |
| GET | `/api/status` | 当前状态（网页也可用 MQTT `arm/web/status`） |
| POST | `/api/control` | `{"action":"start\|pause\|estop\|reset"}` |
| POST | `/api/cmd` | 六关节目标（摄像头脚本主路径） |
| GET | `/api/metrics` | 流量/延迟监测 |
| GET | `/api/metrics/scenarios` | 应用场景清单 |

更完整的协议见 [backend/PROTOCOL.md](backend/PROTOCOL.md)。

## 七、以后接真机 / 服务器时怎么分

| 跑在哪 | 跑什么 |
|--------|--------|
| **云服务器** | Mosquitto（1883/9001）+ FastAPI；`ENABLE_SIMULATOR=false` |
| **比赛电脑** | 摄像头脚本 + Vue 观摩页（连服务器 IP） |
| **SC171V2** | 连服务器 MQTT，订阅 `arm/device/cmd`，上报 status |

电脑上把 `ARM_API_BASE` 改成 `http://服务器IP:8000` 即可。

## 八、常见问题

**1. 页面是 DEMO，不是 SIM**  
后端没连上。先确认 `http://127.0.0.1:8000/api/health`。

**2. 摄像头开着但网页关节乱跳**  
模拟器还开着。用模式 B：`ENABLE_SIMULATOR=false` 后重启后端。

**3. 识别窗口打不开摄像头**  
换索引：改脚本里 `cv2.VideoCapture(0)` 为 `1`；或检查是否被其他软件占用。

**4. 权重警告 / 识别乱**  
确认 `coarse_heavy.h5` 路径；无权重时预测无意义。

**5. MQTT 一直失败**  
默认只用 HTTP，可忽略。要用 MQTT 先安装并启动 Mosquitto，再设 `ARM_USE_MQTT=1`。

**6. 端口被占用**  
结束占用 8000/5173 的进程，或改端口后同步改 `ARM_API_BASE` / Vite proxy。
